## 1. 项目整体概述

### 1.1 项目背景

本项目是**智慧航空加油系统**，是服务于上海浦东机场的核心业务系统之一。系统基于微服务架构设计，采用Apache Dubbo作为RPC框架，Spring Framework作为应用框架，MySQL作为主数据库，Redis作为缓存中间件，Zookeeper作为服务注册中心，Apache Camel和ActiveMQ作为消息中间件。系统主要服务于航空燃油供应业务，实现航班信息接收、加油任务调度、电子签单管理、终端设备通信等核心功能。

系统的核心业务价值在于实现航油供应全流程的数字化管理，从航班落地到加油完成再到电子签单确认，整个流程实现了无纸化操作。系统需要处理浦东机场每天数百架次航班的加油任务，涉及东航、南航、国航、海航等多家航空公司的航班，实时性要求高，数据准确性要求严格。

### 1.2 业务范围

系统涵盖的业务范围包括以下几个方面：

**航班信息管理**：系统通过ESB（企业服务总线）接收航空公司和机场的航班动态信息，包括航班计划、延误信息、变更信息等。这些信息经过处理后推送到终端设备和服务调度人员，为加油任务安排提供数据支撑。航班信息管理模块需要处理航班号、机型、机号、起降时间、登机口、行李转盘等多种信息，并支持多航空公司数据的标准化处理。

**加油任务调度**：这是系统的核心业务功能，包括任务生成、任务分配、任务执行、任务监控、任务归档等全生命周期管理。调度员根据航班信息和可用资源情况进行任务分配，任务下达到终端设备后由加油员执行加油操作。系统支持多种任务分配策略，包括按区域分配、按人员分配、按车辆分配等。

**电子签单系统**：加油完成后，加油员通过终端设备进行电子签名确认，系统生成正式的油单记录。油单包含航班信息、加油量、油品类型、签单人、时间戳等完整信息。电子签单支持多种油品类型（航空煤油、汽油等），支持不同计量单位转换，支持油单审核和作废流程。

**终端设备管理**：系统管理大量终端设备，包括5500终端和PAD终端两种类型。终端设备与后台系统通过RMDP协议和REST协议进行通信，实现任务接收、状态上报、消息推送等功能。终端设备管理包括设备注册、状态监控、固件升级、参数配置等功能。

**数据对接**：系统需要与多个外部系统进行数据交互，包括东航、南航等航空公司的航班系统，机场的行李系统、登机口系统，以及总部的数据平台。对接方式包括WebService、REST API、消息队列等多种形式。

### 2.2 架构特点

本系统的架构设计具有以下几个显著特点：

**微服务与模块化结合**：系统整体采用模块化设计，每个业务模块独立部署但又相互协作。核心业务模块（flight-oil、flightdispatch等）通过Dubbo进行RPC调用，而非核心功能模块（messagemanager、globalsequence等）提供通用服务能力。这种设计既保证了系统的可维护性，又兼顾了部署灵活性。

**多协议通信支持**：系统支持多种通信协议，包括Dubbo RPC用于内部服务调用、REST API用于前端交互、HTTP REST用于外部系统对接、ActiveMQ+JMS用于异步消息处理、Camel路由用于复杂数据流编排。这种多协议支持使得系统能够适应不同的集成场景。

**前后端分离**：智慧航空加油系统前端采用Vue.js + Element-UI构建独立SPA应用，Pudong-Web采用Angular构建多租户平台。前端通过HTTP REST API与后端服务交互，完全解耦。这种架构使得前端可以独立开发和部署，也便于多端适配。

**多租户支持**：系统通过security模块和resourceparent模块实现了完整的多租户体系，支持按租户（航空公司、机场等）进行数据隔离和权限控制。多租户架构使得系统能够同时服务多个客户，降低了运维成本。

**终端适配**：系统设计了terminal-dispatch模块作为消息分发枢纽，将消息同时推送到5500终端和PAD终端两种设备类型。这种设计考虑了不同终端设备的特性差异，实现了统一的消息推送和差异化的终端适配。

## 3. 模块清单与功能说明

### 3.1 模块总览

本项目共包含27个业务模块，涵盖核心业务、终端管理、数据同步、接口对接、安全认证、资源管理等多个领域。以下按功能分类进行详细说明。

### 3.2 核心业务模块

#### 3.2.1 flight-oil（加油业务模块）

flight-oil是系统的核心业务模块，负责航班加油任务的全流程管理。模块包含flight-oil-master（主服务）、flight-oil-service（服务接口）两个子模块。master模块包含70个Java源文件和33个XML配置文件，实现了任务生成、任务分配、任务执行，油单生成等核心业务逻辑。service模块定义了对外暴露的服务接口，供其他模块调用。

**核心功能**：
- **航班信息处理**：接收航班动态，解析航班信息，计算加油需求
- **任务生成**：根据航班信息和油品配置自动生成加油任务
- **任务调度**：将任务分配给具体的加油员和加油车
- **油单生成**：加油完成后自动生成油单，计算加油量和费用
- **任务监控**：实时监控任务执行状态，支持任务催办和取消

**核心类**：
- `OilBillService`：油单服务，处理油单的增删改查
- `OilBillEntity`：油单实体类，定义油单数据结构
- `OilTaskService`：任务服务，处理任务的生成和分配

**依赖模块**：依赖globalsequence（全局序列）、messagemanager（消息管理）、terminalmanager（终端管理）等模块。

#### 3.2.2 flightdispatch（航班调度模块）

flightdispatch是航班调度指挥模块，负责航班信息的展示、调度决策和任务派发。模块包含117个Java源文件，提供航班监控、调度决策、资源调配等功能。该模块是调度员日常工作的主要操作界面。

**核心功能**：
- **航班监控**：实时展示航班动态，支持按航班号、机型、状态等条件筛选
- **调度决策**：根据航班信息和可用资源提供调度建议
- **任务派发**：将调度决策转化为具体的任务并下达到终端
- **进度跟踪**：实时跟踪任务执行进度，异常情况及时预警
- **统计报表**：提供航班统计、加油量统计、效率分析等报表

**核心类**：
- `FlightDispatchController`：调度控制器，处理HTTP请求
- `FlightInfoService`：航班信息服务，提供航班数据查询
- `TaskDispatchService`：任务派发服务，处理任务分配逻辑

**依赖模块**：依赖flight-oil（加油业务）、terminalmanager（终端管理）等模块。

#### 3.2.3 flightdealui（签单UI模块）

flightdealui是电子签单的前端模块，提供签单操作的Web界面。模块包含51个Java源文件和7个HTML模板文件，采用Spring MVC架构。调度员和审核员通过该模块进行油单查看、审核、打印等操作。

**核心功能**：
- **油单列表**：展示待签单、已签单、已审核等状态的油单
- **油单详情**：展示油单详细信息，包括航班信息、加油记录、签单人等
- **油单审核**：审核员对已签单进行审核确认
- **油单打印**：支持油单打印和导出PDF
- **签单查询**：支持按时间、航班号、油品类型等条件查询

**依赖模块**：依赖flight-oil（获取油单数据）、messagemanager（消息通知）等模块。

#### 3.2.4 agdis-oil-web（智慧航空加油系统Web模块）

agdis-oil-web是智慧航空加油系统的核心Web应用模块，提供调度管理界面、消息处理引擎、Dubbo服务暴露和WebSocket实时通信能力。模块采用Maven多模块架构，包含agdis-bean（实体类）、agdis-core（核心业务）、agdis-controller（控制器）、agdis-dubbo（Dubbo服务）、agdis-redis（Redis缓存）、agdis-message-producer（消息生产）、agdis-boot（Camel消息路由）、agdis-websocket（WebSocket通信）、agdis-web-oil（Web应用）共9个子模块，483个Java源文件。

| agdis-bean | 130 | 实体类定义 |
| agdis-core | 159 | 核心业务逻辑 |

| agdis-dubbo | 39 | Dubbo服务 |

**核心功能**：

- **Dubbo服务层**：通过Zookeeper注册，提供航班管理、任务管理、资源管理等RPC服务接口
- **Camel消息路由**：处理8个核心消息队列（Q.FLT.WCDS、Q.CLIENT.FLIGHTTASK、Q.CLIENT.LOGIN等），实现航班、任务、告警等消息的分发
- **WebSocket通信**：基于LMAX Disruptor实现高并发实时消息推送
- **Redis缓存**：提供分布式缓存支持
- **Web管理界面**：基于JSP/Spring MVC的调度管理界面

**核心技术栈**：

- Apache Dubbo 2.5.3 RPC框架
- LMAX Disruptor 3.3.2 高并发处理
- WebSocket实时通信
- Spring MVC 4.1.5

**依赖模块**：依赖globalsequence（全局序列）、messagemanager（消息管理）、security（安全认证）等模块。

### 3.3 终端管理模块

#### 3.3.1 terminalmanager（终端管理模块）

terminalmanager是终端设备管理模块，负责5500终端和PAD终端的通信和管理。模块采用多子模块设计，包括terminal-core（核心逻辑）、terminal-rest（REST服务）、terminal-rmdp（RMDP协议）、terminal-mode（数据模型）、messagedispatch（消息分发）、bsod（基础对象）等6个子模块。模块提供完整的终端适配能力，支持终端登录、任务接收、状态上报、消息推送等交互。

**核心功能**：
- **终端认证**：终端设备通过RMDP协议登录认证，支持设备绑定和用户绑定
- **任务下发**：将加油任务推送到终端设备，支持多种任务类型
- **状态接收**：接收终端上报的任务状态更新，包括开始、进行中、完成等
- **消息推送**：向终端推送航班变更、任务变更等通知消息
- **资源查询**：终端可查询航班信息、油品信息等资源数据

**协议支持**：
- **RMDP协议**：终端设备专用协议，包含登录（RELI）、注销（RELO）、资源初始化（SINI）、任务上报（TRPT）等命令
- **REST API**：提供标准HTTP接口供外部系统调用

**核心类**：
- `MessageDisposer`：消息处理器，处理终端消息的收发
- `TaskInfoDisposer`：任务处理器，处理任务相关逻辑
- `FlightInfoDisposer`：航班信息处理器，处理航班查询和订阅

**依赖模块**：依赖resourceparent（资源服务）、security（安全认证）等模块。

#### 3.3.2 terminalmanager-oil（终端管理油单模块）

terminalmanager-oil是terminalmanager的特化版本，专门处理油单相关的终端业务。模块在terminalmanager基础上增加了油单展示、油单签名、化验单管理等功能，适用于油车终端设备。该模块包含terminal（主模块）、messagedispatch（消息分发）、bsod（基础对象）三个子模块。

**核心功能**：
- **油单展示**：在终端设备上展示当前任务对应的油单信息
- **电子签名**：支持在终端上进行电子签名确认
- **化验单管理**：管理油品化验单，记录化验结果
- **压力差管理**：记录加油前后的压力差数据
- **一机一座**：支持移动终端的一机一座功能认证

**核心类**：
- `TaskService`：任务服务，处理任务相关业务
- `OilBillService`：油单服务，处理油单相关业务
- `OneMachineOneSeatService`：一机一座认证服务

**依赖模块**：依赖flight-oil-service（油单服务）、vdinfo（车辆绑定）等模块。

#### 3.3.3 terminal-dispatch（终端分发模块）

terminal-dispatch是消息分发模块，作为消息路由枢纽将消息同时分发到5500终端和PAD终端。模块采用Apache Camel作为路由引擎，ActiveMQ作为消息队列，实现消息的异步分发和负载均衡。模块配置简单但功能关键，是终端消息推送的核心组件。

**核心功能**：
- **消息路由**：接收来自各业务模块的消息，根据规则路由到对应终端队列
- **消息广播**：支持将同一条消息同时推送到两种终端类型
- **日志记录**：记录消息流转日志便于问题排查
- **配置管理**：支持通过配置文件管理队列名称和路由规则

**核心配置**：
- 输入队列：Q.TERMINAL.WORKMESSAGE（工作消息）、Q.TASK（任务）等
- 输出队列：
  - 5500终端：Q.TERMINAL.WORKMESSAGE_5500、Q.TASK_5500等
  - PAD终端：Q.TERMINAL.WORKMESSAGE_PAD、Q.TASK_PAD等

**依赖模块**：依赖Apache Camel、ActiveMQ等中间件。

### 3.4 数据同步模块

#### 3.4.1 synch-data（数据同步模块）

synch-data是浦东数据同步模块，负责将本地数据同步到总部平台。模块采用Spring Boot 2.0.8架构，包含BILLSettlementManager（油单结算同步）、BILLUploadManager（油单上传）、TaskUploadManager（任务上传）等核心组件。同步采用HTTP REST方式与总部平台通信，支持同步失败重试和数据清理功能。

**核心功能**：
- **油单上传**：将本地生成的油单上传到总部平台
- **任务上传**：将本地任务数据上传到总部平台
- **结算同步**：同步油单结算状态到总部平台
- **失败重试**：上传失败的记录自动重试，支持配置重试次数和间隔
- **数据清理**：定期清理已成功上传的历史数据

**核心类**：
- `SynchTimer`：同步定时器，触发同步任务
- `BILLUploadManager`：油单上传管理器
- `TaskUploadManager`：任务上传管理器
- `IASMessageTransfer`：消息传输组件

**依赖模块**：依赖flight-oil-service（油单数据）、dubbo（远程调用）等模块。

#### 3.4.2 pvg-transfer-oilbill-mdb（油单同步MDB模块）

pvg-transfer-oilbill-mdb是油单同步到MDB数据库的模块，负责将油单数据从本地数据库同步到MDB数据库。模块包含OILSourceManager（源数据管理）、ExtractManager（数据抽取）、TimerManager（定时管理）等组件。模块采用多线程技术提高同步效率，支持增量同步和全量同步两种模式。

**核心功能**：
- **数据抽取**：从源数据库抽取油单数据
- **数据转换**：将源数据转换为MDB格式
- **数据加载**：将转换后的数据加载到MDB数据库
- **状态同步**：同步油单状态变更到MDB
- **日志记录**：记录同步过程日志便于问题排查

**核心类**：
- `OILSourceManager`：源数据管理接口
- `ExtractManager`：数据抽取管理器

#### 3.4.3 pvg-transfer-bill-mdb-dmp（油单同步DMP模块）

pvg-transfer-bill-mdb-dmp是油单同步到DMP数据管理平台的模块，与pvg-transfer-oilbill-mdb类似但目标不同。该模块专注于油单数据的管理和分发，支持按航空公司、按机场等维度进行数据隔离。模块包含OilTaskSheetService（油单服务）、OilBillManager（油单管理）等组件。

**核心功能**：

**依赖模块**：依赖flight-oil（油单数据）等模块。

### 3.5 公共服务模块

#### 3.5.1 globalsequence（全局序列模块）

globalsequence是全局序列生成模块，负责为系统提供全局唯一的序列号。模块采用数据库表存储序列信息，支持多种序列策略（每日重置、连续递增等）。序列号用于油单号、任务号等业务主键的生成。

**核心功能**：
- **序列生成**：提供全局唯一的序列号

**核心类**：
- `ISequenceService`：序列服务接口

**依赖模块**：无强依赖，被其他模块广泛调用。

#### 3.5.2 messagemanager（消息管理模块）

messagemanager是消息管理模块，负责系统内各类消息的存储和分发。模块包含messagemanager_api（API接口）、messagemanager_impl（服务实现）两个子模块。支持消息的创建、发送、接收、查询等操作，通过ActiveMQ实现异步消息处理。

**核心功能**：

#### 3.5.3 servicemanager（服务管理模块）

servicemanager是服务管理模块，负责系统中各类服务（Task、Service）的生命周期管理。模块实现了命令模式，通过ServiceEngine统一管理服务的创建、执行、监控和回收。模块还提供服务标准的配置化管理，支持动态修改服务流程。

### 3.6 安全与资源模块

#### 3.6.1 security（安全模块）

security是安全认证与授权模块，负责系统的用户认证和权限控制。模块基于Apache Shiro实现完整的认证授权体系，支持多租户隔离、细粒度权限控制、会话管理等功能。模块还集成了CAS单点登录支持，可与企业SSO系统集成。

**核心功能**：
- **用户认证**：用户名密码认证、验证码校验
- **权限授权**：基于RBAC的权限模型，支持细粒度控制
- **会话管理**：Session创建、刷新、超时处理
- **多租户支持**：租户数据隔离和租户内权限控制
- **CAS集成**：支持CAS单点登录协议

**核心类**：
- `IAuthService`：认证服务接口
- `AuthService`：认证服务实现
- `SecurityRealm`：Shiro认证域
- `SecurityManager`：安全管理器

**依赖模块**：依赖resourceparent（用户资源）、mis（用户信息）等模块。

#### 3.6.2 resourceparent（资源父模块）

resourceparent是资源管理父模块，负责用户、资源、权限的统一管理。模块包含login（登录服务）、resourceapi（资源API）两个子模块。提供用户认证、资源管理、权限控制等核心功能，是其他模块进行权限控制的基础。

**核心功能**：
- **用户登录**：提供用户登录和登出功能

- **权限控制**：基于资源的细粒度权限控制
- **车辆绑定**：管理用户与车辆的绑定关系
- **在线状态**：管理用户在线状态和超时注销

**核心类**：
- `ILoginWebservice`：登录服务接口
- `IDispatchLoginService`：调度登录服务
- `IUserService`：用户服务接口

## 6. 数据库设计总览

### 6.1 数据库实例

系统涉及多个数据库实例，各数据库承担不同的职责：

| 数据库实例 | 主要用途 | 连接方式 |
|-----------|----------|----------|
| **mis_mutil** | 用户、组织、权限等基础数据 | Druid连接池 |
| **db_oil_pvg** | 航班、油单、任务等业务数据 | Druid连接池 |
| **MDB** | 移动数据库（油单同步目标） | JDBC直连 |
| **DMP** | 数据管理平台（数据同步目标） | JDBC直连 |
| **Redis** | 缓存、会话、分布式锁 | Jedis客户端 |

### 6.2 核心业务表分类

#### 6.2.1 航班相关表

航班相关表用于存储航班信息和航班动态，支撑加油任务调度业务。核心表包括：

- `flight_info`：航班基础信息表，存储航班号、机型、机号、起降时间等
- `flight_dynamic`：航班动态表，存储航班状态变更记录
- `flight_plan`：航班计划表，存储航班计划信息
- `flight_change_log`：航班变更日志表

#### 6.2.2 任务相关表

任务相关表用于存储加油任务和任务执行状态，支撑任务调度和监控业务。核心表包括：

- `oil_task`：加油任务表，存储任务基本信息、状态、关联航班等
- `task_allocation`：任务分配表，存储任务分配给人员和车辆的信息
- `task_log`：任务日志表，记录任务状态变更历史
- `task_monitor`：任务监控表，存储任务实时状态

#### 6.2.3 油单相关表

油单相关表用于存储电子签单数据，支撑油单管理和结算业务。核心表包括：

- `oil_bill`：油单主表，存储油单基本信息、油品信息、金额等
- `oil_bill_detail`：油单明细表，存储加油量、油温、密度等详细信息
- `oil_bill_sign`：油单签名表，存储电子签名数据
- `oil_assay_sheet`：化验单表，存储油品化验结果

#### 6.2.4 用户与权限表

用户与权限相关表用于存储用户、组织、角色和权限数据，支撑安全认证和授权业务。核心表包括：

- `sys_user`：用户表，存储用户账号、姓名、联系方式等
- `sys_role`：角色表，存储角色定义
- `sys_user_role`：用户角色关联表
- `sys_permission`：权限表，存储细粒度权限定义
- `sys_dept`：部门表，存储组织架构
- `sys_tenant`：租户表，存储租户信息

#### 6.2.5 资源与设备表

资源与设备相关表用于存储车辆、设备、人员等资源信息，支撑资源管理和终端通信业务。核心表包括：

- `vehicle_info`：车辆信息表，存储车辆基本信息
- `device_info`：设备信息表，存储终端设备信息
- `vehicle_device_bind`：车辆设备绑定表，存储车辆与设备的关联
- `registered_resource`：注册资源表，存储在线资源状态

#### 6.2.6 消息与日志表

消息与日志相关表用于存储系统消息和操作日志，支撑消息通知和问题排查。核心表包括：

- `system_message`：系统消息表，存储各类业务消息
- `message_record`：消息记录表，存储消息发送和接收记录
- `operation_log`：操作日志表，存储用户操作记录
- `alarm_log`：告警日志表，存储系统告警信息

### 6.3 数据库设计特点

#### 6.3.1 多租户支持

系统通过`tenant_id`字段实现多租户数据隔离，每个租户的数据相互独立。同时支持按租户进行数据权限控制，确保用户只能访问授权范围内的数据。

#### 6.3.2 数据安全设计

核心业务表普遍包含以下安全字段：

- `del_flag`：删除标记，支持逻辑删除
- `create_time`/`create_user_id`：创建时间和创建人
- `update_time`/`update_user_id`：更新时间、更新人
- `version`：版本号，支持乐观锁

#### 6.3.3 索引设计

关键业务表在常用查询字段上建立了索引，提升查询性能：

- `flight_info`：flight_number、flight_date、status
- `oil_task`：task_number、flight_id、status
- `oil_bill`：bill_number、tenant_id、create_time

### 7.1 认证体系

系统采用多层次认证体系，支持多种认证方式：

**用户名密码认证**：通过`security`模块的认证服务实现，支持用户名密码校验、验证码校验。认证成功后生成Token返回给客户端，后续请求携带Token进行身份识别。

**CAS单点登录**：集成CAS协议实现单点登录，与企业SSO系统集成。用户登录企业SSO后可无缝访问本系统。

**终端认证**：终端设备通过RMDP协议进行认证，需要提供设备号和用户凭证。认证通过后建立长连接进行消息通信。

### 7.2 授权体系

系统采用RBAC（基于角色的访问控制）模型进行权限管理：

**用户-角色-权限三层模型**：
- `User`（用户）：登录系统的自然人
- `Role`（角色）：一组权限的集合，代表某种岗位或职能
- `Permission`（权限）：对系统资源的访问许可

**权限控制粒度**：
- **菜单权限**：控制用户可访问的菜单项
- **按钮权限**：控制用户可执行的操作
- **数据权限**：控制用户可访问的数据范围

### 7.3 多租户隔离

系统通过`security`模块和`resourceparent`模块实现完整的多租户体系：

**租户隔离策略**：
- **数据隔离**：通过`tenant_id`字段在物理层面隔离数据
- **权限隔离**：用户只能操作其所属租户的资源
- **配置隔离**：不同租户可拥有独立的业务配置

**租户上下文传递**：
- 请求头中携带`tenant_id`
- Dubbo调用时通过Attachment传递租户上下文
- 线程本地存储当前租户信息

系统采用Shiro的PasswordMatcher进行密码匹配，支持多种密码策略：

## 11. 系统整体风险分析

基于对28个业务模块代码的全面分析，系统存在以下风险点，按严重程度分为高、中、低三个等级。

### 11.1 高风险问题（需立即处理）

#### 11.1.1 框架版本老旧风险

| 风险项 | 影响范围 | 风险等级 | 说明 |
|--------|----------|----------|------|
| **Java 1.7 停止支持** | 全系统 | 🔴 高 | Java 1.7 已停止官方支持，存在安全漏洞，无法使用新特性 |
| **Spring Framework 3.x/4.x 停止维护** | 大部分模块 | 🔴 高 | 存在已知安全漏洞，无官方补丁 |
| **Apache Dubbo 2.5.x 停止维护** | 核心服务 | 🔴 高 | 不支持新特性，功能受限 |
| **Spring Boot 1.5.x 停止支持** | 数据共享平台 | 🔴 高 | 存在安全风险，无法升级依赖 |

**影响分析**：
- 继续使用已停止支持的框架版本，将面临安全漏洞无法修复的风险
- 新发现的CVE漏洞不会得到官方修复
- 与现代工具链、IDE的兼容性逐渐下降

#### 11.1.2 SQL注入风险

| 模块 | 问题位置 | 风险等级 | 严重程度 |
|------|----------|----------|----------|
| flight-oil | 油单Mapper使用${}拼接 | 🔴 高 | 可被SQL注入攻击 |
| messagemanager | 动态SQL构建 | 🔴 高 | 参数校验不足 |
| oil-pvg-data-share | TaskService | 🔴 高 | 存在注入风险 |
| flight-services-parent | 动态HQL构建 | 🔴 高 | HQL注入风险 |
| security | 权限查询SQL | 🔴 高 | 敏感操作存在风险 |

**典型问题代码**：
```xml
<!-- 危险写法 -->
<select id="queryFlight">
    SELECT * FROM flight WHERE flight_no = '${flightNo}'
</select>
```

**优化建议**：
- 全部改用 `#{}` 参数化查询
- 实施SQL白名单校验
- 使用MyBatis参数化查询替代字符串拼接

#### 11.1.3 内存泄漏风险

| 风险项 | 模块 | 触发条件 | 影响 |
|--------|------|----------|------|
| **通知Map内存泄漏** | resourcesub | 员工订阅航班后注销 | 订阅关系无法释放 |
| **缓存无限增长** | Redis缓存 | 大批量数据查询 | 内存耗尽 |
| **类加载器泄漏** | 多模块集成 | 反复热部署 | 堆内存溢出 |

#### 11.1.4 数据一致性风险

| 风险场景 | 涉及模块 | 触发条件 | 后果 |
|----------|----------|----------|------|
| **多终端并发操作** | terminalmanager | 两终端同时处理同一任务 | 数据覆盖、状态混乱 |
| **缓存与数据库不一致** | Redis + MySQL | 缓存未及时更新 | 查询到过期数据 |
| **多模块数据同步延迟** | synch-data, pvg-transfer-* | 同步任务失败 | 各系统数据不一致 |
| **分布式事务不完整** | Dubbo服务调用 | 服务调用链中断 | 部分数据回滚 |

### 11.2 中等风险问题（需近期规划）

#### 11.2.1 架构设计风险

## 11. oil-extend-sys 扩展系统

### 11.1 模块概述

oil-extend-sys 是智慧航空加油系统的扩展服务模块，提供高级调度算法、数据分析、压差监控等增强功能。该模块采用 Spring Boot 2.7.15 构建，独立部署于端口 8779，通过 Dubbo RPC 与核心业务系统集成。

| 属性 | 值 |
|------|-----|
| **项目名称** | oil-extend-sys |
| **版本** | 1.0.0.1-V1 |
| **服务端口** | 8779 |
| **Java版本** | 1.8 |
| **主要用途** | 智能调度算法、数据分析、压差监控 |

### 11.2 技术架构

#### 11.2.1 技术栈

| 类别 | 技术选型 | 版本 |
|------|----------|------|
| **核心框架** | Spring Boot | 2.7.15 |
| **RPC框架** | Dubbo + Zookeeper | 2.6.5 |
| **数据库** | MySQL 5.1.35 + Druid | 1.2.8 |
| **数据源** | Baomidou Dynamic DS | 3.5.0 |
| **持久层** | MyBatis | 2.2.0 |
| **消息队列** | ActiveMQ | 5.7.0 |
| **定时任务** | XXL-Job | 2.4.1 |
| **缓存** | Redis + Lettuce | - |
| **文档生成** | Apache POI 4.1.2 | iTextPDF 5.5.9 |

#### 11.2.2 架构特点

**多数据源动态切换**：模块配置了 6 个数据源，支持在运行时动态切换数据库连接：
- `master` - db_oil_extend（主库）
- `oil_db_pvg` - db_oil_pvg（浦东加油数据）
- `mis_mutil` - mis_mutil（MIS多租户）
- `mycat` - terminal（MyCat分库）
- `mnFlightInterface` - second-research-institute-can（航班接口）
- `mnOilZhiling` - zhiling-nanhang-jy（指令数据）

**智能调度算法集成**：支持匈牙利算法和 VRP（车辆路径规划）算法两种派工策略，通过外部 REST 算法模块实现复杂调度优化。

**完整的定时任务体系**：基于 XXL-Job 实现 11 个定时任务，覆盖数据分析、压差统计、自动派工等业务场景。

### 11.3 核心业务模块

#### 11.3.1 自动派工模块 (AuDis)

自动派工模块是系统的核心功能，实现加油任务的智能分配。

**控制器**：`AuDisController` - 路径前缀 `/audis`

| 接口方法 | 功能说明 |
|----------|----------|
| `canAutomaticDispatch` | 航班设置禁止/允许自动派工 |
| `immediatelyAutomaticDispatch` | 航班设置立即自动派工 |
| `userRefuelStatus` | 加油员设置禁止/允许自动派工 |
| `autoDispatch` | 匈牙利算法自动派工 |
| `vrpAutoDispatch` | VRP算法自动派工 |

**派工模式**：

| 模式 | 代码 | 说明 |
|------|------|------|
| 均衡模式 | `equilibrium` | 任务平均分配给可派工加油员 |
| 效率模式 | `efficiency` | 满足时间窗要求下使用最少加油员 |

**任务状态流转**：
```
TDWN(派发) → TGET(收到) → TACP(接受) → TARV(到位) → TBEG(开始) → TEND(结束)
                ↓                        ↓
            TCAL(取消)              TDEC(拒绝)
```

#### 11.3.2 分析统计模块 (Analysis)

分析统计模块提供加油业务的数据分析和挖掘能力。

**控制器**：`AnalysisController` - 路径前缀 `/analysis`

| 定时任务 | 功能说明 |
|----------|----------|
| `synchronousResourceToAuAnalysisUser` | 同步加油员信息到分析表 |
| `analysisLoadingAndUnloadingTime` | 分析装卸车时间 |
| `analysisRefuelOnTheDay` | 分析当天加油信息 |
| `analysisRefuelOnTheMonth` | 分析当月加油信息 |
| `analysisContinuousRefuelingTime` | 分析连续加油时间 |
| `analysisRestTime` | 分析加油员休息时间 |
| `resetRefuelOnTheDay` | 重置当天加油信息 |
| `analysisAirlineRefuel` | 分析各航线加油信息 |

**分析数据表**：
- `au_analysis_user` - 加油员分析表（装卸车时间、连续工作时间、休息时间等）
- `au_analysis_airline` - 航线分析表（航线加油量、加油时间统计）

#### 11.3.3 压差分析模块 (Pressure)

压差分析模块负责加油过程中压力差的监控和分析。

**控制器**：`PressureController` - 路径前缀 `/pressure`

| 接口方法 | 功能说明 |
|----------|----------|
| `getVehiclePressure` | 获取车辆压差数据 |
| `getPressureAnalysisEchartsData` | 压差分析图表数据 |
| `getPressureChangeEchartsData` | 压差变化趋势数据 |
| `getPressureStatistic` | 压差统计数据 |
| `exportPressureAnalysisData` | 导出压差分析数据 |
| `filterAlert` | 滤芯告警处理 |
| `insertFilterInfo` | 添加过滤器信息 |
| `getFilterInfo` | 查询过滤器信息 |
| `insertFilterMaintenanceSheet` | 添加维护记录 |

**核心功能**：
- 过滤器压差监控与告警
- 压差曲线导出（支持模板配置）
- 滤芯维护保养记录管理

#### 11.3.4 航班管理模块 (Flight)

航班管理模块负责航班信息的查询和管理。

**控制器**：`FlightController` - 路径前缀 `/flight`

| 接口方法 | 功能说明 |
|----------|----------|
| `getViewFlights` | 获取航班视图列表 |

**核心数据表**：`srv_org_flight` - 原始航班信息表

**航班状态**：
- `WAIT_REFUEL` - 等待加油
- `REFUELING` - 加油中
- `REFUELED` - 已完成加油
- `NON_REFUEL` - 无需加油

#### 11.3.5 资源管理模块 (Resource)

资源管理模块负责加油员、车辆等资源的管理。

**管理器实现**：
- `AuResourceManagerTaoImpl` - 桃仙机场资源管理
- `AuResourceManagerHghImpl` - 虹桥机场资源管理
- `AuResourceManagerCgoImpl` - 浦东机场资源管理

**资源类型**：
- 油罐车（VehicleType=1）
- 管线车（VehicleType=2）

#### 11.3.6 消息通知模块 (Message)

消息通知模块基于 ActiveMQ 实现异步消息处理。

**组件**：
- `MessageListener` - 消息监听器
- `MessageManagerImpl` - 消息管理器

**队列配置**：通过 `amq_config` 参数动态配置队列名称。
